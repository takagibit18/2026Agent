# 多 subagent 会议摘要

## 会议目标

围绕以下问题形成统一结论：

- 中国本科大三学生想找大厂 `Agent 开发 / LLM 应用工程师`，应优先掌握哪些技术栈
- 3 个月内最现实、最有求职转化率的学习路径是什么
- 哪些方向应当暂时否决或延后

参会角色：

- `1号研究员`：从岗位真实需求和求职现实出发
- `2号研究员`：从 12 周学习规划和项目落地出发
- `3号研究员`：从招聘经理/面试官评估标准出发
- `主 agent`：组织讨论、提炼共识、输出最终方案

---

## 第一轮独立观点

### 1号研究员核心观点

- 本科大三最现实的入口不是模型算法研究，而是 `AI 应用工程 + 后端工程 + RAG/Tool Calling 落地`
- 关键能力是 `Python、后端、SQL、Linux/Git、RAG、Prompt、Tool Calling`
- 反对把主线过早放在 `微调、复杂多智能体、推理内核、K8s、MCP`

### 2号研究员核心观点

- 应采用 `12 周四阶段` 节奏：基础搭建、项目成型、工程强化、求职冲刺
- 建议形成 `2 主 1 辅` 项目组合
- 强调项目、评测、部署、简历和面试应联动推进

### 3号研究员核心观点

- 招聘经理真正看重的是 `项目可信度、工程能力、评测能力、成本意识、系统设计表达`
- 技术主轴应是 `Python、后端、RAG、Prompt 工程化、Agent Workflow、Eval、Observability`
- 强烈反对 `堆名词、硬冲训练、做万能超级 Agent`

---

## 第二轮交叉评审

### 共识点

3 位研究员全部同意以下判断：

1. 岗位定位必须收敛到 `LLM 应用工程 / Agent 开发 / AI 应用后端`
2. 本科大三不应把主线放在模型训练研究
3. 最有价值的能力组合是 `Python + 后端 + RAG + Tool Calling/Workflow + 评测 + 部署`
4. 项目必须少而深，不能做成堆砌 demo
5. 评测、成本意识、失败分析必须进入最终方案

### 主要争论点

#### 争论点 1：算法刷题权重

- `1号研究员` 认为算法仍是大厂门槛，不能弱
- `3号研究员` 认为算法是保底项，但不能吞掉主线训练时间
- `最终结论`：
  - 算法必须持续并行
  - 但不能压过项目与工程能力

#### 争论点 2：项目数量

- `2号研究员` 提出 `2 主 1 辅`
- `1号研究员` 担心 12 周内做太多会全部做浅
- `最终结论`：
  - 统一表述为 `1 个重项目 + 1 个轻项目/增强模块`
  - 如果时间充裕，可向 `2 主 1 辅` 靠近
  - 但核心原则仍是“少而深”

#### 争论点 3：Agent 的复杂度

- `1号研究员` 和 `2号研究员` 反对过早多智能体
- `3号研究员` 进一步强调要做 `受控 workflow`，而不是自由超级 Agent
- `最终结论`：
  - 只推荐 `单 Agent + Tool Calling + 状态机式编排`
  - 否决复杂多智能体作为 3 个月主线

---

## 被明确否决的方案

3 位研究员全部同意否决以下方向作为本阶段主线：

1. `把大模型训练/微调作为核心路线`
2. `做复杂多智能体系统`
3. `做万能超级 Agent`
4. `以读论文或复现论文作为主要成果`
5. `把 K8s、MCP、推理内核优化放进前排`
6. `只堆框架名词，不做评测与工程闭环`

---

## 最终统一共识

### 统一岗位定位

- `LLM 应用工程师`
- `Agent 开发工程师`
- `AI 应用后端工程师`

### 统一技术主线

按优先顺序收敛为：

1. `Python`
2. `后端服务开发`
3. `SQL + Redis`
4. `RAG`
5. `Prompt + Structured Output + Tool Calling`
6. `Agent Workflow`
7. `评测与实验设计`
8. `成本控制与可观测性`
9. `Linux + Git + Docker`
10. `最小前端展示能力`

### 统一学习路线

- 第 1-3 周：打底与最小闭环
- 第 4-6 周：主项目成型
- 第 7-9 周：工程强化
- 第 10-12 周：求职冲刺

### 统一项目策略

- 以 `1 个重项目 + 1 个轻项目/增强模块` 为核心
- 项目必须具备：
  - 真实场景
  - 工程闭环
  - 评测数据
  - 部署或可演示能力
  - 失败案例与 trade-off 说明

---

## 会议结论

本次多 subagent 讨论最终达成一致：

> 对中国本科大三学生来说，冲大厂 Agent 开发 / LLM 应用工程师的最优路径，是先成为“能把 LLM 做成工程系统的人”，而不是先成为“会很多前沿概念的人”。
